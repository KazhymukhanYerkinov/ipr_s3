## Архитектура

### Clean Architecture — три слоя

```
┌──────────────────────────────────────────────┐
│              Presentation                     │
│   BLoC  ←  Screens  ←  Widgets               │
├──────────────────────────────────────────────┤
│                Domain                         │
│   Entities  ←  Behaviors  ←  Use Cases        │
│   Commands  ←  Strategies                     │
├──────────────────────────────────────────────┤
│                 Data                          │
│   DTOs  ←  Sources  ←  Services  ←  Mappers   │
└──────────────────────────────────────────────┘
```

**Правила зависимостей:**
- `presentation` зависит от `domain`
- `data` реализует `domain`
- `domain` **не зависит ни от кого** — чистые Dart-классы без фреймворков

### Структура проекта

```
lib/
├── main.dart                    ← Точка входа
├── app.dart                     ← MaterialApp.router + BlocProvider<AuthBloc>
├── core/
│   ├── di/                      ← GetIt + Injectable (DI)
│   ├── router/                  ← AutoRoute (навигация)
│   ├── security/                ← EncryptionHelper, SecureLogger, PinManager
│   ├── platform/                ← MethodChannel, FFI
│   ├── collections/             ← CommandEntry (LinkedList)
│   ├── error/                   ← Failures, Exceptions
│   ├── result/                  ← Result<T> (аналог Either)
│   ├── use_case/                ← Callable (базовый UseCase)
│   ├── widgets/                 ← Общие виджеты
│   ├── localization/            ← intl локализация
│   └── extensions/              ← SnackBar extensions
├── features/
│   ├── auth/                    ← Google Sign-In, PIN, биометрия
│   ├── files/                   ← Импорт, шифрование, поиск, сортировка, Undo/Redo
│   ├── folders/                 ← Composite-дерево папок
│   ├── settings/                ← MethodChannel (батарея, storage), тема
│   ├── stats/                   ← Статистика (fold, groupBy, where, take)
│   └── benchmark/               ← Dart vs C FFI, main thread vs Isolate
```

### Ключевые технологические решения

| Решение | Реализация | Почему |
|---------|------------|--------|
| State Management | BLoC + Freezed | Предсказуемость, тестируемость, separation of concerns |
| DI | GetIt + Injectable | Compile-time codegen, ленивая инициализация |
| Routing | AutoRoute | Type-safe, codegen, deep links |
| Local Storage | Hive (encrypted) | Быстрый NoSQL, поддержка AES-256 шифрования |
| Key Storage | flutter_secure_storage | Keychain (iOS) / Keystore (Android) — hardware-backed |
| Error Handling | Result\<T\> (свой) | Читаемее dartz Either, идиоматичный Dart |
| Code Generation | Freezed, json_serializable, injectable, auto_route, hive_generator | Boilerplate elimination |

## Поток работы приложения

```
main.dart
   │
   ├── Firebase.initializeApp()
   ├── Hive.initFlutter() + registerAdapters
   ├── configureDependencies() ← GetIt + Injectable
   ├── Bloc.observer = SecureBlocObserver()
   │
   └── MyApp
        │
        ├── BlocProvider<AuthBloc>
        │     └── AuthCheckRequested → проверяет текущего пользователя
        │
        └── BlocListener<AuthBloc, AuthState>
              ├── Unauthenticated    → AuthSignInScreen (Google Sign-In)
              ├── PinSetupRequired   → SetPinScreen (первый вход)
              ├── PinRequired        → LockScreen (PIN + биометрия)
              └── Authenticated      → HomeScreen (список файлов)
```
