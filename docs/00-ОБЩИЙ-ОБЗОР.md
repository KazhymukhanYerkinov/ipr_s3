# File Secure App — Общий обзор проекта

## Что это за проект?

**File Secure App** — это мобильное приложение для безопасного хранения файлов (фото, PDF, документы) на устройстве пользователя. Все файлы шифруются AES-256 перед сохранением, доступ защищён PIN-кодом и биометрией.

**Ключевая идея**: это как «сейф для файлов» в телефоне — файлы зашифрованы, ключи хранятся в защищённом хранилище (Keychain/Keystore), вход через Google + PIN/биометрия.

---

## Зачем этот проект?

Проект создан для ИПР (Индивидуальный План Развития) в Bank CenterCredit. Он демонстрирует **7 компетенций**, необходимых для уровня **Middle-4**:

| # | Цель | Где реализовано |
|---|------|----------------|
| 1 | Производительность и рендеринг | ListView.builder, RepaintBoundary, BlocSelector, const constructors |
| 2 | CI/CD с GitHub Actions | `.github/workflows/flutter-ci.yaml` — lint, test, build Android/iOS |
| 3 | Isolate и compute | Шифрование через `compute()`, поиск через `Isolate.spawn()` |
| 4 | Безопасность данных | AES-256, SHA-256 PIN, flutter_secure_storage, OAuth 2.0 |
| 5 | Паттерны GoF | Command, Strategy, Factory, Composite |
| 6 | Коллекции Dart | Queue, LinkedList, Set с hashCode, fold/map/where |
| 7 | MethodChannel + FFI | Нативный код (Kotlin/Swift), C-библиотека через FFI |

## Архитектура

### Clean Architecture (3 слоя)

```
┌─────────────────────────────────────────┐
│           PRESENTATION                   │
│   (BLoC, Screens, Widgets)              │
│   - Отвечает за UI и состояние          │
│   - Использует Use Cases                │
├─────────────────────────────────────────┤
│              DOMAIN                      │
│   (Entities, Behaviors, Use Cases)      │
│   - Бизнес-логика                       │
│   - Не зависит от других слоёв          │
├─────────────────────────────────────────┤
│               DATA                       │
│   (DTOs, Sources, Services, Mappers)    │
│   - Работа с данными (Hive, файлы)     │
│   - Реализует Behaviors из Domain       │
└─────────────────────────────────────────┘
```

**Почему Clean Architecture?**
- Разделение ответственности: UI не знает о Hive, Domain не знает о Flutter
- Легко тестировать: можно мокать каждый слой
- Легко менять реализацию: можно заменить Hive на SQLite, не трогая UI

### Стек технологий

| Технология | Зачем |
|-----------|-------|
| **flutter_bloc** | State management — управление состоянием через события и стейты |
| **Freezed** | Кодогенерация immutable классов (state, event) |
| **GetIt + Injectable** | Dependency Injection — внедрение зависимостей |
| **AutoRoute** | Навигация с кодогенерацией маршрутов |
| **Hive** | Локальная база данных (NoSQL, быстрая) |
| **flutter_secure_storage** | Keychain (iOS) / Keystore (Android) для ключей |
| **encrypt** | AES-256 шифрование |
| **dartz** | Either<Failure, T> — функциональная обработка ошибок |
| **Firebase Auth** | Google Sign-In |
| **local_auth** | Биометрия (Face ID / отпечаток) |
| **dart:ffi** | Вызов C-функций из Dart |

## Как работает приложение (User Flow)

```
1. Открытие приложения
   └── main.dart: Firebase.initializeApp() → Hive.initFlutter() → DI → runApp()

2. Проверка авторизации
   └── AuthBloc → AuthCheckRequested → getCurrentUser()
       ├── Нет пользователя → AuthSignInScreen (Google Sign-In)
       ├── Есть, но нет PIN → SetPinScreen
       └── Есть + PIN → LockScreen (ввод PIN / биометрия)

3. Главный экран (HomeScreen)
   └── Список зашифрованных файлов
       ├── Импорт файла → file_picker → шифрование в Isolate → сохранение
       ├── Просмотр → расшифровка в Isolate → показ превью
       ├── Удаление → Command → Undo через SnackBar
       └── Поиск → Isolate.spawn() с двусторонней связью

4. Папки
   └── Древовидная структура (Composite) → FolderTreeScreen

5. Статистика
   └── fold() для общего размера, groupBy по типам, top-5 файлов

6. Настройки
   └── MethodChannel → батарея, диск (Kotlin/Swift нативный код)

7. Бенчмарк
   └── Dart CRC32 vs C CRC32 (FFI) — сравнение скорости
```
