### Код — `encryption_helper.dart`:

```dart
Future<Uint8List> getEncryptionKey() async {
  // ШАГ 1: Пытаемся прочитать существующий ключ
  final existingKey = await _secureStorage.read(key: StorageKeys.encryptionKey);
  if (existingKey != null) {
    return base64Url.decode(existingKey);  // Декодируем base64 → 32 байта
  }

  // ШАГ 2: Ключа нет. Но может данные уже есть? (ключ потерян!)
  if (await _hasExistingEncryptedData()) {
    throw const EncryptionKeyLostException();  // Фатальная ситуация
  }

  // ШАГ 3: Первый запуск — генерируем новый ключ
  final newKey = Hive.generateSecureKey();  // 32 случайных байта
  await _secureStorage.write(
    key: StorageKeys.encryptionKey,
    value: base64Url.encode(newKey),        // Сохраняем как base64 строку
  );
  return Uint8List.fromList(newKey);
}
```

## Часть 2: AES-256-CBC — как работает

### Что такое AES-256?

```
AES = Advanced Encryption Standard
256 = длина ключа в битах (32 байта)

Это СИММЕТРИЧНЫЙ алгоритм:
один и тот же ключ для шифрования И расшифровки

      ┌──────┐                    ┌──────┐
      │ Data │                    │ Data │
      └──┬───┘                    └──▲───┘
         │                           │
    ┌────▼────┐                 ┌────┴────┐
    │ ENCRYPT │                 │ DECRYPT │
    │ AES-256 │                 │ AES-256 │
    └────┬────┘                 └────▲────┘
         │          ┌─────┐          │
         │    ┌────►│ KEY │◄────┐    │
         │    │     └─────┘     │    │
    ┌────▼────┘    (32 байта)   └────┴────┐
    │ Encrypted                  Encrypted │
    │   Data                       Data    │
    └──────────────────────────────────────┘
```

### Что такое CBC (Cipher Block Chaining)?

AES работает с блоками по **16 байт**. CBC — режим, где каждый блок зависит от предыдущего:

```
Данные разбиваются на блоки по 16 байт:

[Block 1 (16 bytes)] [Block 2 (16 bytes)] [Block 3 (16 bytes)] ...

ШИФРОВАНИЕ CBC:

        IV (16 байт, случайный)
        │
        ▼
┌──────────────┐
│  Block 1     │──► XOR ──► AES-256 ──► Encrypted Block 1
│  (plaintext) │    ▲                          │
└──────────────┘    │                          │
                    IV                         │
                                               │
┌──────────────┐                               │
│  Block 2     │──► XOR ──► AES-256 ──► Encrypted Block 2
│  (plaintext) │    ▲                          │
└──────────────┘    │                          │
              Encrypted Block 1                │
                                               │
┌──────────────┐                               │
│  Block 3     │──► XOR ──► AES-256 ──► Encrypted Block 3
│  (plaintext) │    ▲
└──────────────┘    │
              Encrypted Block 2

XOR = побитовое "исключающее или"
Каждый блок XOR-ится с ПРЕДЫДУЩИМ зашифрованным блоком
Первый блок XOR-ится с IV
```

### Зачем IV (Initialization Vector)?

```
БЕЗ IV (режим ECB — небезопасный):

   "hello world 123" ──AES──► 0x7A3F...
   "hello world 123" ──AES──► 0x7A3F...  ← ОДИНАКОВЫЙ результат!
   
   Атакующий видит: два файла одинаковые (даже не зная содержимого)

С РАНДОМНЫМ IV (режим CBC):

   "hello world 123" + IV_1 ──AES──► 0x7A3F...
   "hello world 123" + IV_2 ──AES──► 0xB91C...  ← РАЗНЫЙ результат!
   
   Атакующий не может определить что файлы одинаковые
```

**IV не секретный!** Его задача — обеспечить уникальность шифротекста. Секретный только ключ. Поэтому IV хранится вместе с зашифрованными данными.

### Код шифрования — `encryptBytes()`:

```dart
Uint8List encryptBytes(EncryptionParams params) {
  // 1. Создаём AES-ключ из 32 байт
  final key = enc.Key(params.key);

  // 2. Генерируем СЛУЧАЙНЫЙ IV (16 байт)
  //    Каждый файл получает уникальный IV!
  final iv = enc.IV.fromSecureRandom(16);

  // 3. Создаём шифровальщик: AES-256, режим CBC
  final encrypter = enc.Encrypter(enc.AES(key, mode: enc.AESMode.cbc));

  // 4. Шифруем данные
  final encrypted = encrypter.encryptBytes(params.data, iv: iv);

  // 5. Склеиваем: [IV (16 байт)] + [зашифрованные данные]
  final result = Uint8List(iv.bytes.length + encrypted.bytes.length);
  result.setRange(0, iv.bytes.length, iv.bytes);
  result.setRange(iv.bytes.length, result.length, encrypted.bytes);
  return result;
}
```

### Код расшифровки — `decryptBytes()`:

```dart
Uint8List decryptBytes(EncryptionParams params) {
  // 1. Создаём ключ
  final key = enc.Key(params.key);

  // 2. ИЗВЛЕКАЕМ IV из первых 16 байт
  //    (при шифровании мы положили IV в начало!)
  final ivBytes = params.data.sublist(0, 16);
  final encryptedData = params.data.sublist(16);

  // 3. Создаём IV и шифровальщик
  final iv = enc.IV(ivBytes);
  final encrypter = enc.Encrypter(enc.AES(key, mode: enc.AESMode.cbc));

  // 4. Расшифровываем
  final decrypted = encrypter.decryptBytes(
    enc.Encrypted(encryptedData),
    iv: iv,
  );
  return Uint8List.fromList(decrypted);
}
```

```
                     │                              │
                     ▼                              ▼
              IV = A3 7F 12 8B ...        data = 9E 2C A1 73 ...
                     │                              │
                     └──────────┬───────────────────┘
                                ▼
                    AES-256-CBC decrypt(data, key, iv)
                                │
                                ▼
                    [FF D8 FF E0 00 10 4A 46 ...] (оригинальный JPEG!)
```

## Часть 5: Роль Isolate в шифровании

### Почему нельзя шифровать на main thread?

```
ПЛОХО — шифрование на main thread:

Main Thread:  ──[UI]──[UI]──[████ ШИФРОВАНИЕ 2 сек ████]──[UI]──[UI]──
                              ▲                            ▲
                              │                            │
                         UI ЗАВИСАЕТ                  UI продолжает
                         2 секунды!                   (пользователь видит jank)


ХОРОШО — шифрование в Isolate (compute):

Main Thread:  ──[UI]──[UI]──[UI]──[UI]──[UI]──[UI]──[UI]──[получил результат]──
                       │                                          ▲
                       │    compute()                              │
                       └────────▶ Background ──[████ ШИФРОВАНИЕ ████]──┘
                                  Isolate
                                  (свой heap,
                                   свой thread)
```

### Как compute() работает внутри:

```
Main Isolate                                    Background Isolate
     │                                                │
     │  compute(encryptBytes, params)                  │
     │         │                                       │
     │         ├─── Создаёт новый Isolate ────────────►│
     │         │                                       │
     │         ├─── КОПИРУЕТ params ──────────────────►│  params получены
     │         │    (data: 5MB, key: 32б)              │  (это КОПИЯ, не ссылка!)
     │         │                                       │
     │         │    (main thread свободен,              │  encryptBytes(params)
     │         │     UI работает нормально)             │  ... шифрует ...
     │         │                                       │  ... 1-2 секунды ...
     │         │                                       │
     │         │◄── КОПИРУЕТ результат ────────────────│  return encryptedBytes
     │         │    (encrypted: ~5MB)                   │
     │         │                                       │  Isolate уничтожается
     │         ▼                                       │
     │    Future<Uint8List> completed                   │
     │                                                 X  (dead)
```

### Почему `encryptBytes` — top-level функция?

```dart
// ЭТО РАБОТАЕТ — top-level функция
Uint8List encryptBytes(EncryptionParams params) { ... }

// ЭТО НЕ БУДЕТ РАБОТАТЬ — метод экземпляра
class FileEncryptionService {
  Uint8List _encrypt(EncryptionParams params) { ... }
  // ↑ зависит от this (EncryptionHelper, Logger)
  //   другой Isolate не имеет доступа к this!
}
```

**Правило**: `compute()` принимает **top-level** или **static** функцию. Почему?
- Isolate имеет собственную память (heap)
- Он не видит объекты из main isolate
- Top-level функция не зависит от `this` → может работать в любом Isolate
- Все нужные данные передаём через `EncryptionParams` (data + key)

## Часть 6: Проверка целостности (CRC32 через FFI)

```
ИМПОРТ:
                                         ┌─────────────┐
  encryptedBytes ──→ native_crc32() ──→  │ checksum:   │
                     (C-функция, FFI)    │ 0x3A7F2B1C  │ → сохраняем в Hive
                                         └─────────────┘

РАСШИФРОВКА:
                                         ┌─────────────┐
  encryptedBytes ──→ native_crc32() ──→  │ checksum:   │
                     (C-функция, FFI)    │ 0x3A7F2B1C  │ → сравниваем!
                                         └─────────────┘
                                               │
                                         ┌─────┴─────┐
                                         │           │
                                    совпадает    не совпадает
                                         │           │
                                         ▼           ▼
                                    расшифровка   ОШИБКА!
                                    продолжается  "File integrity
                                                   check failed"
```

**Зачем**: если файл повреждён (битый диск, незавершённая запись) — лучше сообщить об этом, чем выдать мусор после "расшифровки".

## Часть 8: EncryptionQueue (очередь шифрования)

Если пользователь импортирует **несколько файлов** — они обрабатываются по очереди:

```
Пользователь выбирает 3 файла:

  ┌──────────┐  ┌──────────┐  ┌──────────┐
  │ photo1   │  │ photo2   │  │ doc.pdf  │
  │ 5 MB     │  │ 3 MB     │  │ 1 MB     │
  └────┬─────┘  └────┬─────┘  └────┬─────┘
       │              │              │
       ▼              ▼              ▼
  Queue (dart:collection):
  ┌──────────┬──────────┬──────────┐
  │ photo1   │ photo2   │ doc.pdf  │  ← addLast()
  └──────────┴──────────┴──────────┘
       ▲                        
  removeFirst()                  
       │                         
       ▼                         
  _encryptionService.encrypt()    → compute() → Isolate
       │
       ▼
  emit QueueProgress(1/3, 33%)
       │
       ▼
  removeFirst() → photo2 → encrypt → emit(2/3, 66%)
       │
       ▼
  removeFirst() → doc.pdf → encrypt → emit(3/3, 100%, isComplete: true)
```

**Код**:
```dart
final Queue<FileTask> _queue = Queue<FileTask>();

void enqueue(FileTask task) {
  _queue.addLast(task);     // O(1)
  if (!_isProcessing) _processQueue();
}

Future<void> _processQueue() async {
  _isProcessing = true;
  while (_queue.isNotEmpty) {
    final task = _queue.removeFirst();   // O(1), FIFO
    final encrypted = await _encryptionService.encrypt(task.data);
    _progressController.add(QueueProgress(...));
  }
  _isProcessing = false;
}
```

## Итоговая сводка: что → где → как защищено

| Данные | Где хранятся | Как защищены |
|--------|-------------|-------------|
| **AES-256 ключ** | Keychain / Keystore | Hardware-backed шифрование ОС |
| **PIN-код** | Keychain / Keystore | SHA-256 хеш (не plain text) |
| **Зашифрованные файлы** | `secure_files/enc_*` | AES-256-CBC + случайный IV |
| **Thumbnail-ы** | `thumbnails/thumb_*` | AES-256-CBC (тоже шифруем!) |
| **Метаданные** | `secure_files.hive` | Encrypted Hive Box (HiveAesCipher) |
| **Checksum** | Hive Box | CRC32 (проверка целостности) |
