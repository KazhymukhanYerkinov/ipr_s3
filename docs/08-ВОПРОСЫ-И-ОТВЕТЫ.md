# Вопросы и ответы для ассесмента

## Общие вопросы по проекту

### В: Расскажи про проект. Что это?
**О**: File Secure App — приложение для безопасного хранения файлов на устройстве. Пользователь авторизуется через Google Sign-In, устанавливает PIN-код. Файлы шифруются AES-256 перед сохранением — даже если получить доступ к файловой системе, без ключа (который в Keychain/Keystore) файлы не расшифровать. Шифрование происходит в Isolate, чтобы UI не подвисал. Есть папки (древовидная структура), сортировка, поиск, undo/redo, статистика, бенчмарк производительности.

### В: Какая архитектура?
**О**: Clean Architecture с тремя слоями:
- **Presentation** — BLoC для state management, экраны и виджеты
- **Domain** — бизнес-логика: entities, use cases, behaviors (абстрактные интерфейсы репозиториев)
- **Data** — реализация: Hive для хранения, сервисы для шифрования, DTOs

Data реализует behaviors из Domain. Presentation знает только о Domain. Domain не зависит ни от кого.

### В: Почему BLoC, а не Provider/Riverpod?
**О**: BLoC чётко разделяет события (Event) и состояния (State). Каждый Event обрабатывается в handler, каждый State описан через Freezed. Это делает логику предсказуемой и тестируемой — можно протестировать: "при EventX → StateY". Плюс BLoC хорошо интегрируется с Clean Architecture — UseCase вызывается в handler.

### В: Что такое Either<Failure, T>?
**О**: Тип из пакета dartz. Может содержать либо ошибку (Left), либо результат (Right). Вместо throw/catch мы возвращаем:
```dart
// Ошибка
return Left(AuthFailure('Not authenticated'));
// Успех
return Right(user);
```
Преимущества: ошибка — часть сигнатуры функции (не забудешь обработать), код не "летает" через catch, легко тестировать.

**Примечание**: в проекте также используется собственный `Result` класс — суть та же, обёртка для success/failure.

## По архитектуре и коду

### В: Что делает GetIt + Injectable?
**О**: GetIt — Service Locator для DI (Dependency Injection). Injectable — кодогенерация, сканирует аннотации `@injectable`, `@lazySingleton` и генерирует регистрацию зависимостей. Вместо:
```dart
// Без Injectable — ручная регистрация
getIt.registerLazySingleton(() => PinManager(getIt<FlutterSecureStorage>()));
```
Пишем:
```dart
@lazySingleton
class PinManager {
  final FlutterSecureStorage _secureStorage;
  PinManager(this._secureStorage); // Injectable автоматически подставит зависимость
}
```

### В: Как работает AutoRoute?
**О**: AutoRoute генерирует навигацию по аннотациям. Описываешь маршруты в `app_router.dart`, генератор создаёт типизированные Route-классы. Навигация через:
```dart
_appRouter.replaceAll([const HomeRoute()]);
```
Преимущества: типобезопасность (нет строковых путей), автокомплит, генерируемый код.

### В: Как работает auth flow?
**О**:
1. При запуске AuthBloc получает `AuthCheckRequested`
2. Проверяет Firebase User → есть ли активная сессия?
3. Если нет → `Unauthenticated` → экран Sign In (Google)
4. Google Sign-In → Firebase Auth → получаем UserEntity
5. Проверяем PIN → hasPin()
6. Если PIN есть → `PinRequired` → LockScreen (ввод PIN или биометрия)
7. Если PIN нет → `PinSetupRequired` → SetPinScreen
8. После верификации → `Authenticated` → HomeScreen

### В: Как файлы шифруются?
**О**:
1. Пользователь выбирает файл через file_picker
2. Получаем bytes файла
3. Получаем AES-256 ключ из flutter_secure_storage
4. Отправляем bytes + key в `compute()` (Isolate)
5. В Isolate: генерируем случайный IV (16 байт) → шифруем AES-256-CBC → результат = [IV + encrypted_data]
6. Сохраняем зашифрованный файл на диск
7. Метаданные (имя, размер, тип) → в encrypted Hive box
8. Генерируем thumbnail (тоже в Isolate)
9. Считаем CRC32 checksum через FFI (для проверки целостности)

## По теории

### В: Объясни Rendering Pipeline Flutter
**О**: Widget → Element → RenderObject.
- Widget — immutable blueprint. Пересоздаётся при каждом build()
- Element — менеджер. Переиспользуется (canUpdate проверяет тип + key)
- RenderObject — реальная отрисовка: layout (размеры), paint (рисование), hit testing (нажатия)

Когда setState(): Widget пересоздаётся → Element сравнивает → если тот же тип — обновляет RenderObject. Если другой — удаляет старый, создаёт новый.

### В: Что такое Event Loop в Dart?
**О**: Dart однопоточный. Event Loop — бесконечный цикл:
1. Проверяет Microtask Queue (приоритетная) — Future.then, scheduleMicrotask
2. Если пусто → Event Queue — Timer, I/O, UI events
3. Выполняет одну задачу до конца (run-to-completion)
4. Повторяет

async/await — это НЕ многопоточность. Это откладывание в Event Queue. Тяжёлые вычисления всё равно блокируют UI.

### В: Отличие Isolate от Thread?
**О**: Thread — разделяемая память, нужны mutex/lock от race conditions. Isolate — своя изолированная память (heap). Обмен данных только через SendPort/ReceivePort (копированием). Нет race conditions, нет deadlocks. Но копирование данных — overhead.

### В: OWASP Mobile Top 10 — что помнишь?
**О**: M1 — небезопасное использование учётных данных. M3 — слабая аутентификация. M5 — незащищённые коммуникации (нет HTTPS). M6 — утечки данных через логи. M8 — неправильная конфигурация безопасности. M9 — небезопасное хранение данных (SharedPreferences вместо Keychain). В проекте всё покрыто: OAuth 2.0, SecureLogger, flutter_secure_storage, AES-256.

### В: Зачем нужен Dependency Injection?
**О**: Разделение создания и использования зависимостей. Вместо `PinManager(FlutterSecureStorage())` внутри BLoC — инжектим через конструктор. Преимущества:
1. Легко мокать в тестах
2. Один PinManager на всё приложение (Singleton)
3. BLoC не знает как создаётся PinManager — только использует

## Часто задаваемые "каверзные" вопросы

### В: Если всё хранится локально, зачем Firebase Auth?
**О**: Firebase Auth — не для хранения данных, а для **идентификации пользователя**. Google Sign-In даёт:
1. Проверенный email (не нужна верификация)
2. OAuth 2.0 безопасный протокол
3. 2FA на уровне Google
4. Возможность отозвать доступ
5. Не храним пароли — ответственность на Google

### В: Что будет если пользователь удалит приложение и установит заново?
**О**: Все данные потеряются. Ключ шифрования хранится в Keychain/Keystore — при удалении приложения на Android Keystore очищается. На iOS Keychain может сохранить данные, но зашифрованные файлы удалятся с файловой системой.

### В: Зачем так много паттернов? Не over-engineering?
**О**: Каждый паттерн решает конкретную задачу:
- Command — пользователи **реально** случайно удаляют файлы. Undo экономит нервы
- Strategy — 4 варианта сортировки. Без Strategy был бы огромный switch/case
- Factory — 6 типов файлов × разные превью. Один switch вместо разбросанной логики
- Composite — папки **реально** вложенные. Рекурсивный подсчёт без Composite = спагетти-код

### В: Как тестировать BLoC?
**О**: Через bloc_test:
```dart
blocTest<AuthBloc, AuthState>(
  'emits [loading, authenticated] when sign in succeeds',
  build: () => AuthBloc(/* mocked dependencies */),
  act: (bloc) => bloc.add(GoogleSignInRequested()),
  expect: () => [
    AuthState.loading(),
    AuthState.authenticated(user: mockUser),
  ],
);
```
Mock-аем все use cases через mocktail. Проверяем последовательность стейтов.

## Словарь терминов

| Термин | Значение |
|--------|----------|
| **AES-256** | Алгоритм симметричного шифрования с 256-битным ключом |
| **CBC** | Cipher Block Chaining — режим шифрования |
| **IV** | Initialization Vector — случайные данные для уникальности шифротекста |
| **SHA-256** | Криптографическая хеш-функция (односторонняя) |
| **OAuth 2.0** | Протокол авторизации (делегирование доступа) |
| **Keychain** | Безопасное хранилище iOS (hardware-backed) |
| **Keystore** | Безопасное хранилище Android (hardware-backed) |
| **Isolate** | Изолированный поток выполнения со своим heap |
| **compute()** | Однократный запуск функции в Isolate |
| **SendPort/ReceivePort** | Механизм общения между Isolates |
| **FFI** | Foreign Function Interface — вызов C/C++ из Dart |
| **MethodChannel** | Канал связи Flutter ↔ Native (Kotlin/Swift) |
| **BLoC** | Business Logic Component — паттерн управления состоянием |
| **Freezed** | Кодогенерация immutable классов в Dart |
| **GetIt** | Service Locator для Dependency Injection |
| **Hive** | Легковесная NoSQL база данных для Flutter |
| **HiveAesCipher** | Шифрование Hive box с помощью AES |
| **Clean Architecture** | Архитектура с разделением на data/domain/presentation |
| **Either<L, R>** | Тип из dartz: либо ошибка (Left), либо результат (Right) |
| **Graceful degradation** | Приложение работает даже если часть функций недоступна |
| **OWASP** | Open Web Application Security Project — стандарты безопасности |
