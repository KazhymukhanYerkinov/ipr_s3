# Цель 6: Коллекции и структуры данных в Dart

## Теория — обзор коллекций

### Основные коллекции

| Коллекция | Характеристика | Сложность основных операций |
|-----------|---------------|----------------------------|
| `List<T>` | Упорядоченный, индексированный | Доступ O(1), поиск O(n), добавление в конец O(1), вставка O(n) |
| `Set<T>` | Уникальные элементы, неупорядоченный | Добавление O(1), поиск O(1), удаление O(1) |
| `Map<K,V>` | Ключ-значение | Доступ O(1), вставка O(1), удаление O(1) |
| `Queue<T>` | FIFO (First-In-First-Out) | addLast O(1), removeFirst O(1) |
| `LinkedList<T>` | Двусвязный список | add O(1), unlink O(1), доступ по индексу O(n) |

### Mutable vs Immutable

```dart
// Mutable — можно изменять
final list = [1, 2, 3];
list.add(4); // OK

// Immutable — нельзя изменять
final constList = const [1, 2, 3];
constList.add(4); // UnsupportedError!

// UnmodifiableListView — обёртка
final unmodifiable = UnmodifiableListView(list);
unmodifiable.add(4); // UnsupportedError!
// Но! list.add(4) — изменит и unmodifiable (это view, не копия)
```

### hashCode и operator==

В `Set` и `Map` элементы/ключи хранятся в **hash table**. При добавлении:
1. Вычисляется `hashCode` элемента
2. По `hashCode` определяется bucket
3. Внутри bucket сравнивается через `operator==`

```dart
// Если два объекта equal — ОБЯЗАТЕЛЬНО одинаковый hashCode
// Если одинаковый hashCode — НЕ обязательно equal (коллизия)

a == b → a.hashCode == b.hashCode  // ОБЯЗАТЕЛЬНО
a.hashCode == b.hashCode → a == b  // НЕ обязательно
```

### 1. LinkedList — Undo/Redo стек

**Файл:** `lib/core/collections/command_entry.dart`

```dart
import 'dart:collection';
import 'package:ipr_s3/features/files/domain/commands/file_command.dart';

base class CommandEntry extends LinkedListEntry<CommandEntry> {
  final FileCommand command;
  CommandEntry(this.command);
}
```

**Файл:** `lib/features/files/domain/commands/command_manager.dart`

```dart
final LinkedList<CommandEntry> _undoStack = LinkedList<CommandEntry>();
final LinkedList<CommandEntry> _redoStack = LinkedList<CommandEntry>();
```

#### Почему LinkedList, а не List

| Операция | `List<T>` | `LinkedList<T>` | Нужна для Undo? |
|----------|-----------|-----------------|-----------------|
| Добавить в конец | O(1) amortized | **O(1)** | Да (новая команда) |
| Удалить последний | O(1) | **O(1)** | Да (undo) |
| Удалить первый | **O(n)** — сдвиг всех | **O(1)** | Да (лимит 50) |
| Произвольный unlink | O(n) — найти + сдвиг | **O(1)** — unlink | Нет |
| Доступ по индексу | **O(1)** | O(n) | Нет |

Ключевое: при переполнении стека (>50) удаляем первый элемент:

```dart
if (_undoStack.length > _maxHistorySize) {
  _undoStack.first.unlink(); // O(1) у LinkedList, O(n) у List
}
```

`List.removeAt(0)` — **O(n)**, сдвигает все элементы на одну позицию влево.
`LinkedList.first.unlink()` — **O(1)**, просто переключает указатели:

```
До:  [A] ↔ [B] ↔ [C] ↔ [D]
          first           last

A.unlink():
     [B] ↔ [C] ↔ [D]
     first           last
```

#### Как работает LinkedListEntry

```dart
base class CommandEntry extends LinkedListEntry<CommandEntry> {
  // LinkedListEntry предоставляет:
  // - _next: указатель на следующий элемент
  // - _previous: указатель на предыдущий
  // - unlink(): удаляет себя из списка за O(1)
  // - list: ссылка на родительский LinkedList
}
```

`base class` (Dart 3) — можно наследовать, но нельзя реализовать (implement).
Это важно для `LinkedListEntry`, который хранит внутренние указатели.

#### Пример вне проекта

```dart
// Task scheduler с приоритетами
base class TaskNode extends LinkedListEntry<TaskNode> {
  final String name;
  final int priority;
  TaskNode(this.name, this.priority);
}

final tasks = LinkedList<TaskNode>();
tasks.add(TaskNode('backup', 1));
tasks.add(TaskNode('sync', 2));
tasks.add(TaskNode('cleanup', 3));

// Удалить конкретную задачу — O(1)
tasks.first.unlink(); // удалили backup, не сдвигая остальных
```

### 2. Queue — Очередь шифрования

**Файл:** `lib/features/files/data/services/encryption_queue.dart`

```dart
import 'dart:collection';

final Queue<FileTask> _queue = Queue<FileTask>();

void enqueue(FileTask task) {
  _queue.addLast(task);  // добавляем в конец
  _totalEnqueued++;
  if (!_isProcessing) _processQueue();
}

Future<void> _processQueue() async {
  _isProcessing = true;
  while (_queue.isNotEmpty) {
    final task = _queue.removeFirst(); // берём из начала (FIFO)
    final encrypted = await _encryptionService.encrypt(task.data);
    _completedCount++;
    _progressController.add(QueueProgress(...));
  }
  _isProcessing = false;
}
```

#### Почему Queue, а не List

**FIFO** (First-In-First-Out): файлы шифруются **в порядке добавления**.

| Операция | `List<T>` | `Queue<T>` |
|----------|-----------|------------|
| addLast (enqueue) | O(1) | **O(1)** |
| removeFirst (dequeue) | **O(n)** | **O(1)** |
| removeAt(0) | **O(n)** — сдвиг | N/A |

`List.removeAt(0)` сдвигает n-1 элементов:

```
List: [A, B, C, D, E]
removeAt(0):
  [B, C, D, E]  ← B,C,D,E сдвинулись влево = O(n)

Queue (ListQueue, circular buffer):
  [_, _, A, B, C, D, E, _]
         ^head           ^tail
  removeFirst():
  [_, _, _, B, C, D, E, _]
            ^head           = O(1), просто двигаем head pointer
```

#### Под капотом Queue в Dart

`Queue<T>` в Dart реализован как `ListQueue<T>` — **circular buffer** (кольцевой буфер):

```
Массив:  [_, _, A, B, C, D, E, _]
               ^head         ^tail

addLast(F):
          [_, _, A, B, C, D, E, F]
               ^head            ^tail

removeFirst():
          [_, _, _, B, C, D, E, F]
                  ^head         ^tail

Когда tail дойдёт до конца массива — wraps around к началу:
          [G, H, _, B, C, D, E, F]
               ^tail  ^head
```

Когда массив заполняется — создаётся новый 2× размера (amortized O(1)).

#### Семантика

Queue **явно** выражает FIFO. При чтении кода сразу понятно:
- `_queue.addLast(task)` — задача встала в очередь
- `_queue.removeFirst()` — обрабатываем следующую

С `List` это менее очевидно: `list.add(task)` + `list.removeAt(0)`.

#### Progress через Stream

```dart
final _progressController = StreamController<QueueProgress>.broadcast();
Stream<QueueProgress> get progress => _progressController.stream;

// При каждом завершении задачи:
_progressController.add(QueueProgress(
  fileId: task.fileId,
  fileName: task.fileName,
  completedCount: _completedCount,
  totalCount: total,
  isComplete: _queue.isEmpty,
  encryptedData: encrypted,
));
```

UI подписывается на stream и обновляет прогресс-бар.

### 3. fold / map / where / take — Статистика

**Файл:** `lib/features/stats/presentation/bloc/stats_bloc.dart`

```dart
final files = result.value ?? [];

// fold — свернуть список в одно значение
final totalSize = files.fold<int>(0, (sum, f) => sum + f.size);

// Группировка по типу (Map)
final countByType = <FileType, int>{};
final sizeByType = <FileType, int>{};
for (final file in files) {
  countByType[file.type] = (countByType[file.type] ?? 0) + 1;
  sizeByType[file.type] = (sizeByType[file.type] ?? 0) + file.size;
}

// sort + take — топ-5
final recentFiles = [...files]
  ..sort((a, b) => b.createdAt.compareTo(a.createdAt));
final top5Recent = recentFiles.take(5).toList();

final largestFiles = [...files]
  ..sort((a, b) => b.size.compareTo(a.size));
final top5Largest = largestFiles.take(5).toList();
```

#### fold — подробно

```dart
// fold<B>(initialValue, combine) → B
// Обходит список, накапливая результат

files.fold<int>(0, (sum, f) => sum + f.size);

// Раскрытие:
// Шаг 0: sum = 0 (initialValue)
// Шаг 1: sum = 0 + file1.size = 5MB
// Шаг 2: sum = 5 + file2.size = 8MB
// Шаг 3: sum = 8 + file3.size = 11MB
// Результат: 11MB
```

#### fold vs цикл

```dart
// Функциональный стиль (наш выбор)
final totalSize = files.fold<int>(0, (sum, f) => sum + f.size);

// Императивный стиль
var totalSize = 0;
for (final f in files) {
  totalSize += f.size;
}
```

`fold` — **декларативный**: "сверни список в число". Намерение видно сразу.
Цикл — **императивный**: нужно читать тело, чтобы понять что происходит.

#### fold в Composite

```dart
// FolderItem
int get totalSize =>
    children.fold<int>(0, (sum, item) => sum + item.totalSize);
```

`item.totalSize` может быть:
- `FileItem.totalSize` = entity.size (конкретное число)
- `FolderItem.totalSize` = children.fold(...) (рекурсивный вызов!)

fold + Composite = **рекурсивный подсчёт по всему дереву**.

#### where — фильтрация

```dart
// Все изображения
final images = files.where((f) => f.type == FileType.image);

// Файлы больше 1 МБ
final large = files.where((f) => f.size > 1024 * 1024);

// Файлы за последнюю неделю
final recent = files.where(
  (f) => f.createdAt.isAfter(DateTime.now().subtract(Duration(days: 7))),
);
```

#### map — трансформация

```dart
// Имена файлов
final names = files.map((f) => f.name).toList();

// Конвертация в DTO для isolate
final filesData = files.map((f) => {
  'id': f.id,
  'name': f.name,
  'typeIndex': f.type.index,
  // ...
}).toList();
```

#### take — первые N

```dart
// Топ-5 самых больших
final top5 = largestFiles.take(5).toList();
```

`take` — ленивый: не создаёт новый список, а итерирует только первые 5 элементов.
`.toList()` материализует.

### 4. Map — группировка и быстрый доступ

```dart
// Подсчёт файлов по типу
final countByType = <FileType, int>{};
for (final file in files) {
  countByType[file.type] = (countByType[file.type] ?? 0) + 1;
}

// Результат:
// {FileType.image: 15, FileType.pdf: 8, FileType.text: 3}
```

#### Почему Map, а не List + where

```dart
// С Map (O(1) доступ по ключу):
final imageCount = countByType[FileType.image]; // O(1)

// С List + where (O(n) каждый раз):
final imageCount = files.where((f) => f.type == FileType.image).length; // O(n)
```

Если нужно получить count для каждого типа — с Map делаем O(n) один раз,
с List + where — O(n × k), где k = количество типов.

### 5. Осознанный выбор коллекций — сводка

| Задача | Коллекция | Почему НЕ List |
|--------|-----------|----------------|
| Undo/Redo стек | `LinkedList<CommandEntry>` | O(1) unlink первого при лимите 50 |
| Очередь шифрования | `Queue<FileTask>` | O(1) removeFirst (FIFO), семантика очереди |
| Группировка по типу | `Map<FileType, int>` | O(1) доступ по ключу vs O(n) фильтрация |
| Файлы для UI | `List<SecureFileEntity>` | Индексный доступ для GridView.builder |
| Агрегация данных | `fold`, `where`, `take` | Читаемость, декларативность |
| Рекурсия по дереву | `fold` в Composite | Элегантный подсчёт вложенных структур |

### 6. Дополнительные примеры

#### Queue vs List — вставка в начало (Benchmark)

```dart
// Queue — O(1)
final queue = Queue<int>();
for (var i = 0; i < 100000; i++) {
  queue.addFirst(i);  // O(1) — двигаем head pointer
}

// List — O(n) каждая вставка
final list = <int>[];
for (var i = 0; i < 100000; i++) {
  list.insert(0, i);  // O(n) — сдвигаем все элементы
}

// Queue: ~5ms
// List: ~4500ms (в 900× медленнее!)
```

#### Set с кастомным hashCode — теги

```dart
class Tag {
  final String name;
  const Tag(this.name);

  @override
  bool operator ==(Object other) =>
      other is Tag && other.name.toLowerCase() == name.toLowerCase();

  @override
  int get hashCode => name.toLowerCase().hashCode;
}

final tags = <Tag>{};
tags.add(const Tag('Photo'));
tags.add(const Tag('photo'));   // НЕ добавится — дубликат!
tags.add(const Tag('PHOTO'));   // НЕ добавится — дубликат!
print(tags.length);             // 1
```

#### Spread operator и immutable сортировка

```dart
// ❌ Мутирует оригинальный список
files.sort((a, b) => b.size.compareTo(a.size));

// ✅ Создаёт копию, сортирует копию
final sorted = [...files]..sort((a, b) => b.size.compareTo(a.size));
```

`[...files]` — spread operator, создаёт **новый список** (shallow copy).
Оригинальный `files` не изменяется — важно для immutable state в BLoC.
