# Цель 6: Коллекции и структуры данных в Dart

## Теория (что спросят)

### Сравнение коллекций

| Коллекция | Особенность | Вставка | Поиск | Удаление | Когда использовать |
|-----------|------------|---------|-------|----------|--------------------|
| **List** | Упорядоченный, по индексу | O(1) в конец, O(n) в начало | O(n) поиск, O(1) по индексу | O(n) | Когда важен порядок |
| **Set** | Уникальные элементы | O(1) | O(1) contains | O(1) | Когда нужна уникальность |
| **Map** | Ключ → Значение | O(1) | O(1) по ключу | O(1) | Lookup по ключу |
| **Queue** | FIFO (первый вошёл — первый вышел) | O(1) addLast | — | O(1) removeFirst | Очереди задач |
| **LinkedList** | Двусвязный список | O(1) вставка/удаление | O(n) | O(1) если есть ссылка | Stack, частые вставки/удаления |

### Mutable vs Immutable

```dart
// Mutable
final list = [1, 2, 3];
list.add(4); // OK

// Immutable через const
const list = [1, 2, 3];
list.add(4); // ERROR: Cannot add to an unmodifiable list

// UnmodifiableListView
final mutable = [1, 2, 3];
final immutable = UnmodifiableListView(mutable);
```

### hashCode и operator==

Для Set и Map ключ определяется через `hashCode` + `operator==`:
1. Сначала проверяется `hashCode` → если разные, объекты точно разные
2. Если hashCode совпали → вызывается `operator==` для точного сравнения

**Правило**: если два объекта равны (`==`), их `hashCode` **обязан** быть одинаковым.

### 1. LinkedList — Стек Undo/Redo

**Файл**: `lib/core/collections/command_entry.dart`
```dart
// Элемент LinkedList ОБЯЗАН наследовать LinkedListEntry
base class CommandEntry extends LinkedListEntry<CommandEntry> {
  final FileCommand command;
  CommandEntry(this.command);
}
```

**Файл**: `lib/features/files/domain/commands/command_manager.dart`
```dart
class CommandManager {
  // Два стека через LinkedList
  final LinkedList<CommandEntry> _undoStack = LinkedList<CommandEntry>();
  final LinkedList<CommandEntry> _redoStack = LinkedList<CommandEntry>();

  Future<void> execute(FileCommand command) async {
    await command.execute();
    _undoStack.add(CommandEntry(command));  // O(1) — добавление в конец
    _redoStack.clear();

    if (_undoStack.length > 50) {
      _undoStack.first.unlink();  // O(1) — удаление первого элемента
    }
  }

  Future<void> undo() async {
    final entry = _undoStack.last;  // O(1) — доступ к последнему
    entry.unlink();                  // O(1) — удаление из списка
    await entry.command.undo();
    _redoStack.add(CommandEntry(entry.command));
  }
}
```

**Почему LinkedList, а не List?**
- `List.removeAt(0)` — это **O(n)**, потому что все элементы сдвигаются
- `LinkedList.first.unlink()` — это **O(1)**, просто переставляются указатели
- Для стека Undo/Redo мы часто удаляем с обоих концов — LinkedList идеален

**Как объяснить**: "LinkedList из dart:collection — двусвязный список. Каждый элемент знает предыдущий и следующий. Удаление — O(1), потому что просто переставляются ссылки. В List удаление из начала — O(n), потому что все элементы сдвигаются. Для Undo стека с лимитом (удаляем старые с начала) — LinkedList оптимальнее."

### 2. Queue — Очередь шифрования

**Файл**: `lib/features/files/data/services/encryption_queue.dart`
```dart
class EncryptionQueue {
  final Queue<FileTask> _queue = Queue<FileTask>();  // dart:collection

  void enqueue(FileTask task) {
    _queue.addLast(task);   // O(1) — добавляем в конец
    if (!_isProcessing) {
      _processQueue();
    }
  }

  Future<void> _processQueue() async {
    _isProcessing = true;
    while (_queue.isNotEmpty) {
      final task = _queue.removeFirst();  // O(1) — берём из начала
      final encrypted = await _encryptionService.encrypt(task.data);
      _progressController.add(QueueProgress(
        fileId: task.fileId,
        completedCount: _completedCount,
        totalCount: total,
        isComplete: _queue.isEmpty,
        encryptedData: encrypted,
      ));
    }
    _isProcessing = false;
  }
}
```

**Почему Queue, а не List?**
- Queue — FIFO: `addLast()` + `removeFirst()` — оба **O(1)**
- List: `add()` — O(1), но `removeAt(0)` — **O(n)**
- Для очереди задач FIFO — Queue оптимальнее

**Как объяснить**: "Queue из dart:collection реализует FIFO. Файлы добавляются через addLast (в конец очереди) и обрабатываются через removeFirst (из начала). Оба O(1). Если бы использовали List — removeAt(0) был бы O(n), потому что все элементы сдвигаются."

### 3. Set с кастомным hashCode — Теги

**Файл**: `lib/core/collections/tag.dart`
```dart
class Tag {
  final String name;
  const Tag(this.name);

  @override
  bool operator ==(Object other) =>
      identical(this, other) ||
      other is Tag && name.toLowerCase() == other.name.toLowerCase();

  @override
  int get hashCode => name.toLowerCase().hashCode;

  @override
  String toString() => name;
}
```

**Как это работает:**
```dart
final tags = <Tag>{};
tags.add(Tag('Photo'));
tags.add(Tag('photo'));   // НЕ добавится! "Photo" == "photo"
tags.add(Tag('PHOTO'));   // НЕ добавится!
tags.add(Tag('Document')); // Добавится

print(tags.length); // 2: {Photo, Document}
```

**Почему кастомный hashCode?**
- По умолчанию `hashCode` — это identity hash (каждый объект уникален)
- Нам нужно: `Tag('Photo') == Tag('photo')` — case-insensitive сравнение
- Переопределяем `operator==` и `hashCode` (оба обязательно!)
- `hashCode` должен быть одинаковым для равных объектов → `name.toLowerCase().hashCode`

**Как объяснить**: "Tag переопределяет operator== для case-insensitive сравнения. hashCode тоже основан на lowercase name — это обязательно, иначе Set не сможет находить дубликаты. Результат: Set<Tag> автоматически исключает дубликаты тегов, даже если регистр разный."

### 4. fold / map / where — Статистика

**Файл**: `lib/features/stats/presentation/bloc/stats_bloc.dart`

```dart
// fold — агрегация (общий размер всех файлов)
final totalSize = files.fold<int>(0, (sum, f) => sum + f.size);

// Map — группировка по типам
final countByType = <FileType, int>{};
final sizeByType = <FileType, int>{};
for (final file in files) {
  countByType[file.type] = (countByType[file.type] ?? 0) + 1;
  sizeByType[file.type] = (sizeByType[file.type] ?? 0) + file.size;
}

// sort + take — топ-5 самых больших файлов
final largestFiles = [...files]
  ..sort((a, b) => b.size.compareTo(a.size));
final top5Largest = largestFiles.take(5).toList();

// sort + take — топ-5 последних файлов
final recentFiles = [...files]
  ..sort((a, b) => b.createdAt.compareTo(a.createdAt));
final top5Recent = recentFiles.take(5).toList();
```

**fold() — подробно:**
```dart
// fold<int>(initialValue, combine)
// Начинаем с 0, для каждого файла прибавляем его size
files.fold<int>(0, (sum, file) => sum + file.size);

// Эквивалент:
var sum = 0;
for (final file in files) {
  sum += file.size;
}
```

### 5. fold() в Composite — рекурсивный подсчёт

**Файл**: `lib/features/folders/domain/models/folder_item.dart`
```dart
@override
int get totalSize =>
    children.fold<int>(0, (sum, item) => sum + item.totalSize);
```

Если children содержит другой FolderItem — вызовется его totalSize, который тоже fold()... **рекурсия**!

```
Folder "Documents" (totalSize = ?)
├── FileItem "photo.jpg" (size: 5MB)     → totalSize = 5MB
├── FileItem "doc.pdf" (size: 2MB)       → totalSize = 2MB
└── Folder "Backup" (totalSize = ?)
    ├── FileItem "old.jpg" (size: 3MB)   → totalSize = 3MB
    └── FileItem "old.pdf" (size: 1MB)   → totalSize = 1MB

Folder "Backup": fold(0, 3+1) = 4MB
Folder "Documents": fold(0, 5+2+4) = 11MB
```

## Сводка: коллекция → задача

| Задача | Коллекция | Почему |
|--------|-----------|--------|
| Стек Undo/Redo | LinkedList | O(1) удаление с обоих концов |
| Очередь шифрования | Queue | FIFO, O(1) addLast/removeFirst |
| Теги без дубликатов | Set<Tag> | Автоматическая уникальность, O(1) contains |
| Группировка по типам | Map<FileType, int> | O(1) доступ по ключу |
| Подсчёт общего размера | fold() | Агрегация без мутируемой переменной |
| Топ-5 файлов | sort() + take() | Цепочка операций |

## Возможные вопросы на ассесменте

**В**: Когда использовать Queue вместо List?
**О**: Когда нужна FIFO семантика (первый вошёл — первый вышел) и частые операции с обоих концов. Queue — addLast/removeFirst O(1). List — add O(1), но removeAt(0) O(n).

**В**: Зачем нужен кастомный hashCode?
**О**: По умолчанию каждый объект имеет уникальный hashCode (identity). Если мы хотим чтобы Set считал Tag('Photo') и Tag('photo') одинаковыми — нужно переопределить hashCode (одинаковый для равных объектов) и operator==.

**В**: Что будет если переопределить operator== но не hashCode?
**О**: Set и Map не будут работать корректно. Set использует hashCode для бакетов — если hashCode разный, Set даже не вызовет operator==. Два "равных" объекта попадут в разные бакеты.

**В**: Чем fold отличается от reduce?
**О**: fold имеет начальное значение и может вернуть другой тип. reduce не имеет начального значения и работает только с тем же типом. fold([]) на пустом списке вернёт [], reduce на пустом — ошибка.

**В**: Почему LinkedList, а не List для стека?
**О**: Стек Undo имеет лимит (50 команд). При переполнении удаляем самый старый (первый). LinkedList.first.unlink() — O(1). List.removeAt(0) — O(n). При частых удалениях из начала LinkedList эффективнее.
